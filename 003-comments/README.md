# Learning Summary

## Introduction
This program introduces me to comments in C++. It shows me how to write notes inside the code that are not read by the compiler. Comments are useful for explaining what the code does and for leaving reminders for myself or others who may read the program later.

## What I Learnt

### Single line comments in C++
I learnt that two forward slashes `//` are used to create a single line comment. Everything written after `//` on that line is ignored by the compiler. In this program I wrote:

```cpp
// This line will be ignored.
```
This shows that the text following `//` is not executed and only serves as a note.

### Multi line comments in C++

I also learnt that `/*` and `*/` are used to create multi line comments. Everything written between these symbols is ignored, even if it spans several lines. In this program I wrote:
```cpp
/* This line,
as well as this line
will be ignored.*/
```
This is helpful when I want to comment out longer explanations or temporarily remove several lines of code.

### Why comments are important
I learnt that comments do not affect how the program runs but they make the code easier to understand. Good comments can explain the purpose of a line or a block of code, which is especially helpful when looking back at old programs or when working with others.

### Compilation and Execution

The steps for running the program were the same as before:

1. Write the code in a `.cpp` file
2. Compile the code using a C++ compiler
3. Run the program to see the output

The program printed only the line that was not commented out. The comments themselves did not appear in the output.

## Conclusion
With this program, I learnt the following key ideas about comments in C++:

- Single line comments start with `//`
- Multi line comments are written between `/*` and `*/`
- Comments are ignored by the compiler and do not change how the program runs
- Comments are useful for explaining code and improving readability

This gives me a foundation for writing clearer and more understandable programs.
